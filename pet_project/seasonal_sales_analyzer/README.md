## 📝 Описание проекта

Проект анализирует продажи продуктов питания и отвечает на вопросы:
- Что лучше продается летом? 🌞
- Что лучше продается зимой? ❄️
- Какие продукты востребованы круглый год?
- **Как спрогнозировать продажи на будущее?** 🔮
- **Какая маржинальность по категориям и сезонам?** 💰

### 🎯 Возможности:
- ✅ Генерация реалистичных данных о продажах
- ✅ Анализ сезонности по категориям
- ✅ Топ продуктов для каждого сезона
- ✅ Визуализация трендов
- ✅ Рекомендации по ассортименту
- ✅ **ML прогнозирование продаж** (3 модели)
- ✅ **Анализ цен и маржинальности**

## 🔧 Модули

### 1. `data_generator.py`
Генерация реалистичных данных о продажах с учетом:
- Сезонности продуктов
- Праздничных дней
- Выходных эффектов
- Региональных особенностей

### 2. `seasonal_analyzer.py`
Базовый анализ сезонных продаж:
- Анализ по сезонам
- Топ продукты
- Визуализации
- Рекомендации

### 3. `sales_predictor.py` 🆕
ML прогнозирование продаж:
- **3 модели**: Linear Regression, Random Forest, Gradient Boosting
- **Признаки**: временные, лаговые, категориальные
- **Метрики**: MAE, RMSE, R², MAPE
- **Прогноз** на произвольное количество дней

### 4. `margin_analyzer.py` 🆕
Анализ цен и маржинальности:
- Расчет себестоимости и маржи
- Маржинальность по категориям
- Маржинальность по сезонам
- Динамика цен
- Рекомендации по оптимизации

## 🤖 ML Прогнозирование

### Использование:

```python
from sales_predictor import SalesPredictor

# Создаем предиктор
predictor = SalesPredictor(df)

# Обучаем модели
results = predictor.train_models(target_col='revenue')

# Делаем прогноз на 30 дней
forecast = predictor.forecast_future_sales(periods=30)
```

### Результаты обучения (пример):

| Модель | MAE | RMSE | R² | MAPE |
|--------|-----|------|-----|------|
| Linear Regression | 5,115 | 7,319 | 0.7049 | 38.23% |
| Random Forest | 4,106 | 6,147 | 0.7918 | 24.89% |
| **Gradient Boosting** | **4,070** | **6,054** | **0.7980** | **25.36%** |

### Важность признаков:

1. **price** (60.3%) - цена продукта
2. **revenue_ma_7** (30.8%) - скользящее среднее за 7 дней
3. **quantity_lag_7** (2.5%) - количество 7 дней назад
4. **day_of_year** (2.3%) - день года
5. **is_weekend** (1.7%) - выходной день
